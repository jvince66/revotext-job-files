# files.revotext.com

Office-side, M365-SSO-gated file browser for court reporter turn-ins.

**URL pattern:** `https://files.revotext.com/jobs/{JOB_ID}` — e.g. `https://files.revotext.com/jobs/C-60716-002`.

## What it does

1. Office staff opens `files.revotext.com/jobs/C-60716-002` (link from the Reporter Pay board on monday.com).
2. Page loads MSAL.js; user signs in with M365 popup if not already signed in.
3. Page calls `/api/list-files.php?job=C-60716-002` with the Graph access token.
4. `list-files.php` validates the token (via the shared support-portal `auth-helper.php`), checks the M365 email is on the allowlist, validates the job ID pattern, then calls S3 `listObjectsV2` on `jobs/C-60716-002/`.
5. For each object, mints a presigned GET URL (15-min TTL).
6. Renders an HTML table with download links.

## Files in this directory

- `public/jobs/index.php` — the browser-facing HTML page (renders auth flow + table)
- `public/api/list-files.php` — the AJAX JSON endpoint
- `public/index.html` — landing page
- `public/.htaccess` — rewrite `/jobs/{ID}` to `/jobs/index.php?job={ID}`
- `api/aws-config.example.json` — template for the S3 read credentials
- `apache/files.revotext.com.conf` — port-80 Apache vhost
- `apache/files.revotext.com-le-ssl.conf` — port-443 vhost (auto-generated by certbot on the server; recorded here for reference)

## Wire-up on server

`public/api/list-files.php` includes `/var/www/files.revotext.com/api/auth-helper.php`, which is a **symlink** to `/var/www/html/api/auth-helper.php` on the support portal. That way both apps share the same MSAL config + allowlist. PHP's `__DIR__` follows the symlink to the real path, so `db-config.json` and `users.json` are read from the support portal's location.
