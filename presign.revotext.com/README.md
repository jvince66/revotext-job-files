# presign.revotext.com

Anonymous, browser-callable endpoint that hands back short-lived presigned S3 upload URLs to reporters.

## Wire-up

Reporter opens the assignment worksheet at `https://assignments.revotext.com/…` and clicks Upload. That page's JavaScript hits:

```
GET https://presign.revotext.com/?job=C-60716-002&filename=worksheet.pdf
```

and gets back:

```json
{
  "uploadUrl": "https://revotext-portal-documents.s3.amazonaws.com/jobs/C-60716-002/worksheet.pdf?X-Amz-...",
  "expiresIn": 900,
  "key": "jobs/C-60716-002/worksheet.pdf",
  "method": "PUT",
  "maxBytes": 524288000
}
```

The reporter's browser then does `PUT uploadUrl` with the file body directly to S3. S3 accepts the object because the presigned URL was signed by an IAM user with `s3:PutObject` on `jobs/*`.

## Defenses (browser-side anonymous means no secret-based auth)

- **Job ID regex** — rejects anything not matching `^C-\d{3}[A-Z0-9]{2}-\d{3}T?$`
- **Filename sanitization** — no `/`, no `..`, no null bytes, whitelist charset `[A-Za-z0-9._- ()]`, max 255 chars
- **Rate limit** — 60 requests/min per IP, file-based counter under `/tmp/presign-rl/`
- **CORS** — locked to `https://assignments.revotext.com` (also enforced at the S3 bucket)
- **Presigned URL** — PUT only, 15-min TTL, single-key
- **Audit log** — every issuance logged via `error_log()` to `/var/log/apache2/presign-error.log`

## Files

- `public/index.php` — the endpoint (Apache serves `index.php` as directory index)
- `api/aws-config.example.json` — template for the S3 write credentials
- `apache/presign.revotext.com.conf` — port-80 Apache vhost
- `apache/presign.revotext.com-le-ssl.conf` — port-443 vhost (auto-generated by certbot)
