# presign.revotext.com

Server-to-server endpoint that hands back short-lived, presigned S3 upload URLs.

## Wire-up

The reporter's browser NEVER talks to this endpoint. The reporter's worksheet page at
`https://assignments.revotext.com` sends the upload request to its OWN backend; that
backend calls us server-to-server with a shared-secret bearer token:

```
GET https://presign.revotext.com/?job=C-60716-002&filename=worksheet.pdf
Authorization: Bearer <shared-secret>
```

Response:

```json
{
  "uploadUrl": "https://revotext-portal-documents.s3.amazonaws.com/jobs/C-60716-002/worksheet.pdf?X-Amz-...",
  "expiresIn": 900,
  "key": "jobs/C-60716-002/worksheet.pdf",
  "method": "PUT",
  "maxBytes": 524288000
}
```

The caller returns the `uploadUrl` to whichever party will actually PUT the file
(their backend or the reporter's browser). S3 accepts the object because the URL
was signed by our IAM user with `s3:PutObject` on `jobs/*`.

## Defenses

- **Shared-secret bearer token** (required) — `Authorization: Bearer <secret>`; rejected otherwise. Stored in `api/aws-config.json` → `endpoint_secret`.
- **Job ID regex** — rejects anything not matching `^C-\d{3}[A-Z0-9]{2}-\d{3}T?$`
- **Filename sanitization** — no `/`, no `..`, no null bytes, whitelist charset `[A-Za-z0-9._- ()]`, max 255 chars
- **Rate limit** — 300 requests/min GLOBAL (keyed by secret hash, not IP; caller is a single trusted origin so per-IP made no sense)
- **CORS** — remains locked to `https://assignments.revotext.com` for any browser-side fallback path
- **Presigned URL** — PUT only, 15-min TTL, single-key
- **S3 versioning** — enabled on the bucket; a re-uploaded key preserves the prior version
- **Audit log** — every issuance logged via `error_log()` to `/var/log/apache2/presign-error.log`

## Secret rotation

Every 90 days:

1. `openssl rand -base64 36 | tr -dc 'A-Za-z0-9' | head -c 48`  (on Lightsail)
2. Save to `/root/migration-secrets/presign_endpoint_secret` and update `api/aws-config.json`
3. Share new secret with the caller out-of-band (phone; never email/chat)
4. After the caller confirms rollover, revoke the old secret

## Files

- `public/index.php` — the endpoint (Apache serves `index.php` as directory index)
- `api/aws-config.example.json` — template for IAM keys + shared secret
- `apache/presign.revotext.com.conf` — port-80 Apache vhost
- `apache/presign.revotext.com-le-ssl.conf` — port-443 vhost (auto-generated by certbot)
